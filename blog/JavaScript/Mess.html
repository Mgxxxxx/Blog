# Element.ScrollIntoView

在论坛项目中，想实现一个功能：点击发帖，然后页面滚动到底部的输入框。翻阅资料发现了一个挺好用的APIScrollIntoView。这个API接受一个参数，首先是布尔值，默认不穿参的时候布尔值为true。该参数代表元素的顶端和其所在的滚动区的可视区对齐；false代表元素的底部和其所在的滚动区的可视区对齐。若参数为一个对象，则该对象有以下属性可以选择：

  • behavior（可选）： 定义动画过渡效果， "auto"或 "smooth" 之一。默认为 "auto"。
  • block（可选）：定义垂直方向的对齐， "start", "center", "end", 或 "nearest"之一。默认为 "start"。
  • inline（可选）：定义水平方向的对齐， "start", "center", "end", 或 "nearest"之一。默认为 "nearest"。

以下是一个简单的实例：

var element = document.getElementById("box");

element.scrollIntoView();
element.scrollIntoView(false);
element.scrollIntoView({block: "end"});
element.scrollIntoView({behavior: "smooth", block: "end", inline: "nearest"});

# Iterator

# Iterator的出现

在ES6中，我们会经常使用到"for of"这个循环来遍历数组或者对象。"for of"实际上，是使用到了Iterator来实现的。Iterator的出现也是为了提供一个遍历集合的统一机制。

# Iterator的实现

要让每个对象都能使用到Iterator，那么这个对象必须要部署了[Symbol.iterator]方法。并且其必须要含有next方法，return和throw是可选的。如果[Symbol.iterator]方法对应的不是遍历器生成函数（即会返回一个遍历器对象），解释引擎将会报错。若是要能让"for of"使用，next函数的返回值还要是一个包含value和done的对象(done的值为false时，可以没有value属性)。根据以上，我们可以简单自定义一个迭代器。

const a = {
  [Symbol.iterator]() {
    let count = 5;
    return {
      next() {
        let res;
        if (count >= 0) {
          res = { value: count--, done: false };
        } else {
          res = { value: undefined, done: true };
        }
        return res;
      },
    };
  },
};
const it = a[Symbol.iterator]();
for (let r of a) {
  console.log(r);
}

# 调用场合

  • 最常见的就是"for of"
  • 解构赋值：let [first, ...rest] = new Set().add('a').add('b').add('c');
  • 扩展运算符：[..."hello"]
  • yield*，yield*后面跟的时一个可以遍历的解构
  • promise.all()、promise.race()

# 函数参数和函数局部变量名重名问题

看一道题：

var foo = {n:1};
(function (foo) {
  console.log(foo.n);
  foo.n=3;
  var foo = {n:2};
  console.log(foo.n);
})(foo)
console.log(foo.n);

我一开始的想法是第5行使用了var声明，所以函数内部的foo会变量提升，初始值为undefined。所以打印出来的是"undefined 2 1"。可是结果是"1 2 3"。经过调试发现，第5行使用了var声明的和参数名一样的变量，但是并不会重新创建一个变量，foo还是指向参数的foo，所以打印的结果为"1 2 3"。
综上，在函数中使用var声明一个和参数名字一样的变量，声明出来的变量指向的就是相同名字的那个参数。
