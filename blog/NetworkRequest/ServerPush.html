# 服务端推送

为什么需要服务端推送呢？笔者认为，某些情况下，页面需要实时的数据，如何取得实时的数据呢？首先会想到，要取得实时的数据，就要访问服务器，那么我们可以轮询服务器，以此获得最新的数据。接下来就介绍一下实现这种功能的技术Comet

# Comet

Comet指的是一种更高级的Ajax技术。Ajax是一种从页面向服务器请求数据的技术，而Comet则是一种服务器向页面推送数据的技术。

​ ——摘自《JavaScript高级程序设计》

Comet有两种实现方式：长轮询和流。

# 长轮询

介绍长轮询之前，先介绍一下短轮询。短轮询就是浏览器定时向服务器发送请求，看数据有没有更新。而长轮询则是页面先发送一个请求，服务器在没有数据发送的情况下，则一直保持连接，直到有数据可以发送，才会发送响应，结束连接。笔者认为，这其实也还是一种Ajax技术，它只是使用Ajax对服务器进行轮询。

# HTTP流

与短轮询和长轮询不同的是，HTTP流在页面的整个生命周期只使用一个HTTP连接。即浏览器向服务器发送一个请求，但是服务器保持连接打开，周期性的往浏览器发送数据。服务端实现HTTP流的关键是，服务器语言可以将输出缓存一次性发送到客户端上，类似于flush()的功能。而前端实现只要是依靠readyState。下面是客户端实现的一个例子：

var xhr = new XMLHttpRequest();
received = 0;    //最新消息在响应消息的位置
xhr.onreadystatechange = function(){
    if(xhr.readyState == 3){
        result = xhr.responseText.substring(received);
        console.log(result);
        received += result.length;
    }else if(xhr.readyState == 4){
        console.log("完成消息推送");
    }
}
xhr.open('get', url);
xhr.send(null);

以上是浏览器先向服务器发送请求，那么有没有服务器直接推送的呢？答案是有的，以下介绍SSE和WebSocket。

# SSE

SSE(Server-sent Events，服务器发送事件)。它是围绕Comet交互推出的一个API。SEE API会创建服务器的单向连接，服务器可以在这个连接上发送数据。以下是一个简单的例子：

//创建一个EventSource对象，并传入入口点
var source = new EventSource(url);
//建立连接时触发
source.onopen = function(){console.log("open")};
//从服务器接收到新事件时触发
source.onmessage = function(e){
    var data = e.data;
    //处理数据
}
//无法建立连接时触发
source.onerror = function(){console.log("error")};
//关闭连接
source.close();

服务器响应的MIME类型一定要为text/event-stream。响应文本时纯文本。以下是一个简单的响应文本：

data: foo

data: bar

# WebSocket

若通信仍使用HTTP，则HTTP的瓶颈问题就无法彻底解决，故出现了一个新的协议：WebSocket协议。该协议能让浏览器和服务器之间全双工通信。以下列WebSocket的主要特点：

  • 推送功能：服务器可以直接发送数据给浏览器
  • 减少通信量：建立起WebSocket连接后，就一直保持连接状态，减少了连接的开销。WebSocket的首部信息很少，通信量也减少了

# HTTP/2.0

最后介绍一下HTTP/2.0的优点

  • 允许多路复用：多路复用允许同时通过单一的HTTP/2连接发送多重请求-响应信息。改善了：在http1.1中，浏览器客户端在同一时间，针对同一域名下的请求有一定数量限制（连接数量），超过限制会被阻塞。
  • 二进制分帧：HTTP2.0会将所有的传输信息分割为更小的信息或者帧，并对他们进行二进制编码
  • 首部压缩
  • 服务端推送
