# 在setup获取prop,this,route,router,store

# setup中获取prop

setup接受两个参数，一个是props，另一个是context。故我们可以在setup中添加第一个参数props，即可在setup中使用props。

WARNING

但是，因为 props 是响应式的，不能使用 ES6 解构，因为它会消除 prop 的响应性。

若想解构的话，可以使用toRefs来解构props

import { toRefs } from 'vue'

setup(props) {
	const { title } = toRefs(props)

	console.log(title.value)
}

# setup中获取当前实例

# setup第二个参数context

setup中第二个参数是context，它是一个普通的对象，故可以对它使用解构。context上有三个属性：attrs,slots和emit。实际应用中，并不经常使用这些属性。

# getCurrentInstance方法

另一种获取当前实例的方法是getCurrentInstance。获取的实例和从setup中的context获取的实例比较，可以获得更的属性。

此外，getCurrentInstance只能在setup函数或者生命周期函数中使用，若想在其他函数中使用，应使用变量获取实例后，通过访问变量来访问当前实例。

const MyComponent = {
  setup() {
    const internalInstance = getCurrentInstance() // works

    const id = useComponentId() // works

    const handleClick = () => {
      getCurrentInstance() // doesn't work
      useComponentId() // doesn't work

      internalInstance // works
    }

    onMounted(() => {
      getCurrentInstance() // works
    })

    return () =>
      h(
        'button',
        {
          onClick: handleClick
        },
        `uid: ${id}`
      )
  }
}

// also works if called on a composable
function useComponentId() {
  return getCurrentInstance().uid
}

# setup中获取route和router

在setup中可以通过getCurrentInstance来访问route和router。但是，项目打包上线后，我发现，会报错(访问store也会报错)。之后我便使用useRoute和useRouter来在setup中获取这两者，打包上线后，错误消失了。

import { useRouter, useRoute } from 'vue-router'

export default {
  setup() {
    const router = useRouter()
    const route = useRoute()

    function pushWithQuery(query) {
      router.push({
        name: 'search',
        query: {
          ...route.query,
        },
      })
    }
  },
}

# setup中获取store

由于setup不能访问this，所以我们可以直接在初始化store的地方，导出store，然后在需要使用的地方从这个文件导入store。

// /src/store/index.js
import {
  createStore
} from 'vuex'

export default createStore({
  state: {},
  mutations: {},
  actions: {},
  modules: {}
})

// other.js
import store from "/src/store/index.js"
...
setup(){
    //use store...
}
