# Webpack五个基本的概念

  • 入口（entry）：webpack处理的入口，我们可以把要使用到的依赖放到入口文件中，webpack会分析入口文件所需要的依赖，然后会取解析这些依赖。我们可以在webpack配置中配置entry属性。默认为./src。
  • 出口（output）：webpack会将整个应用编译到output指定的文件中。我们可以在webpack配置中配置output属性。默认为./dist。
  • loader：loader可以说是各种文件的编译器，它可以将非JS文件转化为webpack能够处理的模块。我们可以在module.rules中进行配置。在webpack的配置中loader有两个目标：
    • test 属性，用于标识出应该被对应的 loader 进行转换的某个或某些文件。
    • use 属性，表示进行转换时，应该使用哪个 loader。
  • 插件（plugins）：loader 被用于转换某些类型的模块，而插件则可以用于执行范围更广的任务。插件的范围包括，从打包优化和压缩，一直到重新定义环境中的变量。插件接口功能极其强大，可以用来处理各种各样的任务。想要使用一个插件，你只需要 require() 它，然后把它添加到 plugins 数组中
  • 模式：通过选择development或production之中的一个，来设置mode参数，你可以启用相应模式下的webpack内置的优化

# Webpack开发环境基本配置

const {
  resolve
} = require('path');
const HtmlWebpackPlugin = require('html-webpack-plugin');
const {
  CleanWebpackPlugin
} = require('clean-webpack-plugin');

module.exports = {
  mode: "development",  //模式设置为开发模式
  entry: "./src/index.js", //入口文件
  output: {
    filename: "built.js", //设置输出文件的文件名
    path: resolve(__dirname, "dist"),  //设置输出文件的目录
    // clean: true  //webpack5中可以配置该字段使得在构建前清除output目录
  },
  module: {
    rules: [{   //使用sass-loader解析sass文件
        test: /\.sass$/,
        /**
         * 解析顺序为：
         * 使用sass-loader解析为css
         * 使用css-loader整合到js文件中
         * 使用style-loader将css添加到style标签中
         */
        use: ['style-loader', 'css-loader', 'sass-loader']
      },
      {   //使用css-loader解析css文件
        test: /\.css$/,
        use: ['style-loader', 'css-loader']
      },
      {   //使用url-loader解析后缀为png、jpg、gif的图片文件
        test: /\.(png)|(jpg)|(gif)$/,
        loader: 'url-loader',
        options: {
          limit: 8 * 1024,  //若图片大小小于8KB的则转化为base64格式
          name: '[hash:10].[ext]',  //设置打包后的文件名，[hash:10]表示十位哈希值，[ext]表示原来的后缀名
          // esModule: false
          outputPath: "img/"
        }
      },
      {   //使用html-loader来解析html文件中对图片src的引用问题
        test: /\.html$/,
        loader: 'html-loader',
        options: {
          esModule: false,  //关闭es6模块方式
        }
      },
      { //test中匹配到的文件则使用file-loader解析
        test: /\.(woff|woff2|eot|ttf|otf|svg)$/,
        loader: 'file-loader',
        options: {
          name: '[hash:10].[ext]',
          outputPath: "media/",  //打包后输出的位置
        }
      }
    ]
  },
  plugins: [
    new HtmlWebpackPlugin({   //该插件能够使用自己的html模板
      template: "./src/index.html",
    }),
    new CleanWebpackPlugin(), //该插件能够在生成最终文件前将原先的文件清空
  ],
  devServer: {  //简单的web服务器
    contentBase: resolve(__dirname, "dist"),  //内容文件位置
    compress: true, //启动gzip压缩
    port: 8080, //设置可以通过哪个端口访问
    open: true  //启动时是否自动开启浏览器
  }
}

# css文件处理

# 提取css为单文件

首先要下载min-css-extract-plugin插件。下面是简单的配置:

const MiniCssExtractPlugin = require('mini-css-extract-plugin');

module:{
  rules:[
      {
        test: /\.css$/,
        use: [
          {
            loader: MiniCssExtractPlugin.loader,
            options: {
              publicPath: "/dist"   //注意要使用这个属性，否则会路径不正确
            }
          },
          'css-loader'
        ]
      },
  ],
  plugins: [
    new MiniCssExtractPlugin({
      filename: "css/built.css" //文件输出的位置和文件名
    })
  ],
}

# css兼容性

首先要下载postcss-loader和autoprefixer。 简单配置如下

process.env.NODE_ENV = "development";
module: {
  rules:[
    {
      test: /\.css$/,
      use: [
        // 'style-loader',
        {
          loader: MiniCssExtractPlugin.loader,
          options: {
            publicPath: "/dist"
          }
        },
        "css-loader",
        {
          loader: "postcss-loader",
          options: {
            ident: 'postcss',
            plugins: () => [
              require('autoprefixer')()
            ]
          }
        }
      ]
    },
  ]
}
//package.json
//这里设置对浏览器的兼容
"browserslist": {
  "development": [
    "last 1 chrome version",
    "last 1 firefox version",
    "last 1 safari version"
  ],
  "production": [
    ">0.2%",
    "not dead",
    "not op_mini all"
  ]
}

# js文件处理

# js兼容性

首先下载babel-loader@8.0.0-beta.0、@babel/core、@babel/preset-env、core-js,配置如下

module.exports = {
  module: {
    {
      test: /\.js$/,
      exclude: /(node_modules|bower_components)/,
      use: {
        loader: "babel-loader",
        options: {
          presets: [
            [
              "@babel/preset-env",
              {
                useBuiltIns: 'usage', //按需加载
                corejs: {
                  version: 3
                }, //指定corejs版本
                targets: {
                  chrome: '60',
                  firefox: '60',
                  ie: '9',
                  safari: '10',
                  edge: '17'
                }
              }
            ]
          ]
        }
      }
    },
  }
}

# HMR（Hot Module Replacement）

启动HRM功能，设置module.exports.devServer.hot为true。若使用webpack5，则需要设置module.exports.target为web。以下是简单的配置：

const webpack = require('webpack');
module.exports: {
  plugins: [
    new webpack.HotModuleReplacementPlugin()
  ]
  devServer: {
    hot: true
  },
  target: 'web'
}

# source map

为了更容易地追踪错误和警告，JavaScript提供了source map功能，将编译后的代码映射回原始源代码。如果一个错误来自于b.js，source map就会明确的告诉你。

具体的source map使用方法可见https://www.webpackjs.com/configuration/devtool/ (opens new window)，视频讲解可见https://www.bilibili.com/video/BV1e7411j7T5?p=21 (opens new window)

# tree shaking

如果某些文件导出的未被使用，例如a.js道出了function b和function c，但是c未被使用，但是最后会被打包出来，增加打包后的体积。我们可以通过设置package.json中的sideEffects来让c不被打包。如果所有代码都不包含副作用，我们就可以简单地将该属性标记为false。如果你的代码确实有一些副作用，那么可以改为提供一个数组，例如css、@babel/polyfill文件。设置为"sideEffects":["*.css"]。

# 单页面应用代码分割

我们需要对output和optimization属性进行配置，如下：

module.exports: {
  output: {
    //filename属性必须要有一个[name]的占位符
    filename: "js/[name].[contenthash:10].built.js",
    path: resolve(__dirname, "dist")
  },
  optimization: {
    //splitChunks用于将node_modules中使用到的代码抽离出来为一个chunk
    splitChunks: {
      chunks: "all",
      minSize: 20 * 1024, //分割的chunk最小为20kb
    }
  }
}

# 懒加载和预加载

//若不设置`webpackPrefetch`则为懒加载
//设置了`webpackPrefetch:true`则为预加载
//预加载是在所有资源加载完成后再加载的
document.getElementById('btn').addEventListener('click', () => {
  import( /* webpackChunkName:'print', webpackPrefetch: true */ "./print.js").then(({
    print
  }) => {
    print();
  })
})

# module.exports.resolve

通过这个属性我们可以简化我们一些工作

module.exports = {
  resolve: {
    alias:{
      //配置路径别名，例如/src/css可以被映射为$css
      $css: resolve(__dirname, "src/css")
    },
    //配置省略文件路径的后缀名
    extensions: ['.js','.json']
  }
}
